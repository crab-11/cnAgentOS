## 一、技术栈与约束

### 技术栈

| 组件名称 | 版本 | 用途 | 本地路径 | 状态 |
|---------|------|------|---------|------|
| Tornado | 6.5.5 | Python Web 服务框架 | Python 依赖 | ✅ 已使用 |
| SQLite3 | Python 内置 | 轻量级数据库 | `database/app.db` | ✅ 已使用 |
| Layui | 2.13.6 | 后台管理 UI | `app/static/libs/layui-v2.13.6/` | ✅ 后台已使用 |
| Bootstrap | 5.3.8 | 用户侧响应式布局/后续页面备用 | `app/static/libs/bootstrap-5.3.8-dist/` | ✅ 已本地安装 |
| Font Awesome | 5.15.4 | 图标库 | `app/static/libs/fontawesome-free-5.15.4-web/` | ✅ 后台已使用 |

### 开发限制

1. 遵循前置任务开发成果，保证功能、页面风格和代码结构一致。
2. 后台管理页面优先使用 Layui。
3. 第三方前端资源必须本地化引用，禁止 CDN。
4. 后端遵循 MVC 分层：Controller、Model、Template 分离。
5. 写操作接口必须支持 XSRF 防护。
6. 默认系统用户 `admin` 必须受到保护。

## 五、验收要点

### 当前已完成模块验收

1. 访问 `/admin/auth/login`，使用 `admin/admin888` 登录后台。
2. 后台登录页应为浅色沉浸式登录界面。
3. 用户管理页可以新增普通用户，填写昵称、邮箱、密码并设置角色。
4. 用户管理列表应显示用户名、昵称、邮箱、角色、状态、最后登录时间、创建时间。
5. 普通用户可以修改、删除、批量删除。
6. `admin` 用户只能修改密码，不能修改用户名、状态、角色，不能删除。
7. 角色管理中系统角色不能修改和删除。
8. 普通角色可以新增、修改、删除、启用/禁用，并在新增/编辑时勾选功能权限。
9. 功能管理可以按树形结构维护功能分组和菜单节点。
10. 删除按钮在操作列中应完整显示，不被省略号遮挡。
11. 后台页面引用 Layui 和 Font Awesome 本地文件，不引用互联网 CDN。
12. 数据仓库深采成功后，列表中应显示明确的详情入口，而不仅是状态标记。
13. 深采详情页应可查看 AI 摘要、关键词、实体识别和正文摘录。
14. 后台应可管理数字员工名称、别名、分类、模型/API 绑定关系。
15. 默认数字员工应至少包含 `川小码`、`天气`、`音乐` 三项。
16. 数字员工测试应支持 AI 流式响应和 API 同步响应两种方式。
